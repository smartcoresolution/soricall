# SoriCall 서비스 시나리오 및 화면 일치성 검토 — 얼굴·화상 확인 보완본

> 서비스명: 안심소리 가족콜  
> 앱 이름: SoriCall  
> 문서 목적: 스마트폰 Web/App 화면과 서비스 시나리오의 일치성 정리 및 “가족 얼굴 사진 + 화상 가족 확인 + 얼굴 비교” 흐름 추가  
> 수정일: 2026-06-26

---

## 1. 결론: 가능한가?

부분적으로 가능합니다.

다만 사용자가 말한 **“전화를 건 분의 전화기에서 화상통화로 자동 연결”**은 구현 방식에 따라 가능 여부가 달라집니다.

| 구분 | 가능 여부 | 설명 |
|---|---:|---|
| 일반 전화망 통화 중, 상대방 휴대폰 카메라를 앱이 임의로 켜서 자동 화상 연결 | 불가에 가까움 | 상대방 기기에 SoriCall이 설치되어 있지 않거나 동의하지 않으면 카메라·마이크·영상 연결을 시작할 수 없습니다. |
| 가족도 SoriCall 앱을 설치하고 로그인한 상태에서, 의심 상황 발생 시 가족에게 화상 확인 요청 발송 | 가능 | Push 알림 또는 앱 내 알림으로 가족에게 “화상 확인 요청”을 보내고, 가족이 수락하면 WebRTC 기반 화상 연결을 시작할 수 있습니다. |
| 가족이 수락한 화상 화면에서 등록된 가족 얼굴과 실시간 영상 얼굴 비교 | 가능 | 사전 동의, 얼굴 사진 등록, 카메라 권한, 얼굴 특징값 보관 정책이 필요합니다. |
| 어르신 앱 화면에 기존 등록 얼굴과 실시간 가족 영상 얼굴을 나란히 표시 | 가능 | “등록된 얼굴”과 “화상통화 얼굴”을 비교 표시하는 UI로 구현 가능합니다. |
| 얼굴 비교 결과만으로 보이스피싱 확정 판단 | 비권장 | 조명, 카메라 품질, 네트워크 상태, 가족 대리 응답 등 오탐 가능성이 있어 “참고용 확인 신호”로 사용하는 것이 안전합니다. |

따라서 SoriCall에서는 **무단 자동 연결**이 아니라 다음과 같은 방식이 적합합니다.

```text
의심 전화 발생
→ 어르신이 “화상 가족 확인 요청” 누름
→ 등록된 가족/보호자에게 알림 전송
→ 가족이 SoriCall 앱에서 수락
→ 앱-투-앱 화상통화 연결
→ 등록 얼굴과 실시간 얼굴 비교
→ 결과를 어르신/보호자에게 참고용으로 표시
→ 최종적으로 저장된 가족 번호로 재통화 권장
```

## 2. 보완된 전체 서비스 흐름

```text
보호자 회원가입
        ↓
부모님 프로필 생성
        ↓
부모님 스마트폰에 SoriCall 설치
        ↓
QR 코드 또는 초대코드로 가족 연결
        ↓
가족 연락처 등록
        ↓
가족 얼굴 사진 등록
        ↓
안심 단어 설정
        ↓
가족 음성 샘플 등록
        ↓
의심 전화 발생
        ↓
번호 위험도 판단
        ↓
어르신에게 의심 전화 경고
        ↓
안심 단어 확인 또는 보호자 알림
        ↓
필요 시 화상 가족 확인 요청
        ↓
가족이 화상 확인 수락
        ↓
등록 얼굴과 실시간 영상 얼굴 비교
        ↓
통화 후 음성 샘플 분석
        ↓
위험 이력 저장
```

## 시나리오 3. 가족 얼굴 사진을 등록하는 경우

### 목적

어르신과 보호자가 의심 전화 상황에서 실제 가족을 직관적으로 확인할 수 있도록 가족 얼굴 사진을 등록합니다.

| 단계 | 사용자 | 화면 | 내용 |
|---:|---|---|---|
| 1 | 보호자/가족 | 가족 등록 및 목록 | 가족별 얼굴 사진 등록 선택 |
| 2 | 가족 | 얼굴 사진 등록 | 카메라 촬영 또는 사진 선택 |
| 3 | 시스템 | 얼굴 프로필 생성 | 등록 얼굴 이미지 또는 얼굴 특징값 저장 |
| 4 | 시스템 | 가족 목록 표시 | 가족 카드에 얼굴 사진 표시 |
| 5 | 어르신 | 홈/가족 보기 | 가족 얼굴을 보고 쉽게 구분 |

### 개인정보 원칙

```text
얼굴 사진은 가족 확인 목적에만 사용
얼굴 비교 기능 사용 전 별도 동의 필요
삭제 기능 제공
얼굴 특징값은 암호화 저장 권장
```

## 시나리오 8. 화상 가족 확인 및 얼굴 비교를 하는 경우

### 목적

상대방이 가족이라고 주장하지만 어르신이 불안할 때, 가족에게 SoriCall 화상 확인 요청을 보내고, 등록된 얼굴과 실시간 영상 얼굴을 비교합니다.

### 전제 조건

| 조건 | 설명 |
|---|---|
| 가족도 SoriCall 설치 | 화상 확인을 받는 가족도 앱 설치 및 로그인 필요 |
| 가족 얼굴 사전 등록 | 등록 얼굴 사진 또는 얼굴 특징값 필요 |
| 카메라/마이크 권한 | 화상통화 시작 전 권한 허용 필요 |
| 명시적 수락 | 가족이 화상 확인 요청을 수락해야 연결 |
| 동의 로그 | 얼굴 사진, 화상 확인, 얼굴 비교에 대한 동의 기록 필요 |

### 상세 흐름

| 단계 | 사용자/시스템 | 내용 |
|---:|---|---|
| 1 | 어르신 | 의심 전화 경고 화면에서 “화상 가족 확인 요청” 선택 |
| 2 | 시스템 | 현재 전화번호가 등록 가족 번호인지 확인 |
| 3 | 시스템 | 해당 가족 또는 대표 보호자에게 Push 알림 전송 |
| 4 | 가족 | SoriCall 앱에서 “화상 확인 수락” 선택 |
| 5 | 시스템 | 앱-투-앱 화상 연결 시작 |
| 6 | 시스템 | 등록된 가족 얼굴과 실시간 영상 얼굴을 나란히 표시 |
| 7 | AI | 얼굴 일치 가능성 산출 |
| 8 | 앱 | “일치 가능성 높음/낮음/확인 필요”로 표시 |
| 9 | 보호자 | 실제 통화 여부 응답 |
| 10 | 어르신 | 저장된 가족 번호로 다시 전화하도록 안내 |

### 화면 문구 예시

```text
가족에게 화상 확인 요청을 보냈습니다.
가족이 수락하면 얼굴을 확인할 수 있습니다.
```

```text
등록된 얼굴과 현재 영상 얼굴을 비교 중입니다.
결과는 참고용입니다. 최종 확인은 저장된 가족 번호로 다시 전화하세요.
```

### 얼굴 비교 결과 표시 예시

| 결과 | 앱 표시 | 권장 행동 |
|---|---|---|
| 일치 가능성 높음 | 가족 얼굴과 비슷합니다 | 그래도 송금·앱 설치 요청은 보호자에게 확인 |
| 일치 가능성 낮음 | 등록 얼굴과 다를 수 있습니다 | 통화 종료 후 저장된 번호로 재통화 |
| 확인 어려움 | 조명/화질 때문에 확인이 어렵습니다 | 보호자에게 알림 및 재통화 권장 |
| 가족이 수락하지 않음 | 화상 확인이 연결되지 않았습니다 | 통화 종료 후 저장된 번호로 재확인 |

### 금지/주의 사항

```text
상대방 전화기의 카메라를 무단으로 켜면 안 됨
가족이 수락하지 않은 화상 연결은 하지 않음
얼굴 비교 결과를 “본인 확정”으로 표현하지 않음
얼굴 사진과 영상 데이터는 민감정보로 취급
저장 기간, 삭제, 동의 철회 기능 제공
```

## 시나리오 9. 보호자가 의심 전화 알림을 받는 경우

| 단계 | 사용자 | 화면 | 내용 |
|---:|---|---|---|
| 1 | 시스템 | 보호자 알림 | “부모님이 의심 전화를 받고 있습니다” |
| 2 | 보호자 | 보호자 확인 | 부모님 얼굴, 전화번호, 위험도, 시간 확인 |
| 3 | 보호자 | 얼굴 참조 | 등록된 가족 얼굴과 상황 확인 |
| 4 | 보호자 | 응답 버튼 선택 | 내가 전화함 / 내가 아님 / 확인 어려움 |
| 5 | 시스템 | 응답 저장 | 위험 이벤트에 보호자 응답 기록 |
| 6 | 어르신 앱 | 안내 표시 | 사칭 의심이면 강한 경고 표시 |

### 보호자 버튼

```text
[내가 전화함]
[내가 아님, 사칭 의심]
[확인 어려움]
[화상 확인 수락]
```

## 6.3 DB 테이블 추가 후보

```sql
CREATE TABLE face_profiles (
```

## 7. 개인정보·보안 보완

얼굴 사진과 얼굴 특징값은 민감도가 높은 정보이므로 다음 정책이 필요합니다.

| 항목 | 정책 |
|---|---|
| 명시적 동의 | 얼굴 사진 등록 전 별도 동의 |
| 목적 제한 | 가족 확인 및 보이스피싱 예방 목적 외 사용 금지 |
| 최소 보관 | 원본 이미지 보관 최소화, 특징값 중심 저장 검토 |
| 암호화 | 저장 시 암호화, 전송 시 TLS |
| 삭제권 | 가족 얼굴 프로필 삭제 기능 제공 |
| 접근통제 | 관리자 접근 최소화 및 감사로그 |
| 결과 표현 | 얼굴 비교 결과를 확정 신원 인증처럼 표현하지 않음 |
| 화상 연결 | 양쪽 사용자 수락 후 연결 |

## 8. MVP 반영 범위 제안

## 8.1 1차 MVP

```text
가족 얼굴 사진 등록
가족 목록/보호자 알림/분석 결과 화면에 얼굴 표시
의심 전화 시 “화상 확인 요청” 버튼 제공
```

```text
WebRTC 앱-투-앱 화상 연결
```

## 9. 최종 요약

```text
SoriCall의 얼굴·화상 확인 기능은 “자동으로 상대방 카메라를 켜는 방식”이 아니라,
가족이 SoriCall 앱에서 화상 확인 요청을 수락한 뒤
등록된 얼굴과 실시간 영상 얼굴을 비교하는 동의 기반 방식으로 구현하는 것이 적합합니다.
```

**추천 표현**

> 의심 전화가 오면 SoriCall이 가족에게 화상 확인 요청을 보내고,  
> 가족이 수락하면 등록된 얼굴과 실시간 얼굴을 비교해  
> 가족 사칭 가능성을 한 번 더 확인합니다.
