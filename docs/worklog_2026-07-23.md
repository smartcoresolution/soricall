## 가족 등록 흐름

- 로그인 후 기존 가족·초대·음성·얼굴 등록상태를 조회해 최초 설정 화면과 안심 홈을 구분하도록 정리했다.
- 본인 전화 보호용 확인 가족과 부모님 전화 보호용 가족 데이터가 섞이지 않도록 조회·화면 전환을 보완했다.
- 동일 가족에게 등록 링크를 반복 생성할 때 미완료 초대를 재사용하고 목록에 중복 카드가 쌓이지 않도록 수정했다.
- SOLAPI 자동 발송 대신 휴대전화 문자 앱에서 사용자가 직접 링크를 전송하는 흐름으로 통일했다.
- 다른 앱 공유, 링크 복사, 새 창 열기 등 중복 전달 수단을 문자 앱 전송 중심으로 정리했다.
- 초대받은 가족의 휴대전화 번호가 등록 번호와 일치하면 별도 개발용 인증번호 없이 음성 등록으로 이동하도록 단순화했다.
- 전화번호 불일치 오류를 한국어로 표시하고 번호 재확인 및 다음 음성·얼굴 등록 단계를 안내하도록 개선했다.

## 음성·얼굴 등록

- 음성 프로필 생성 시 초대 ID를 동의 로그 ID로 잘못 저장해 발생하던 PostgreSQL 외래키 500 오류를 수정했다.
- 공개 초대 등록의 동의 근거는 감사 로그에 남기고, 기존 음성·얼굴 프로필의 중복 생성을 방지했다.
- 음성 등록 직후 초대를 완료하던 흐름을 `음성 녹음 → 얼굴 촬영 → 최종 완료` 순서로 변경했다.
- SoriCall의 얼굴 등록을 선택사항에서 필수사항으로 변경했다.
  - 회원가입 얼굴정보 처리 동의 필수
  - 화면의 선택 문구 및 건너뛰기 버튼 제거
  - API의 `face_image_ref` 필수 검증
- 잘못 완료된 dave 초대는 기존 음성을 유지한 채 얼굴 등록 전 상태로 복구했다.
