## 1. 현행 구현 개요

SoriCall은 휴대전화 기반 가입, 보호 대상과 확인 가족 구성, 음성·얼굴·기기 등록, Android 수신 전화 선별, AI 다중 신호 위험 분석, 가족 확인 및 긴급 알림을 하나의 모노레포에서 제공한다.

주요 구성:

- Web/PWA: React, TypeScript, Vite, Playwright
- Android: Kotlin, Jetpack Compose, `CallScreeningService`, FCM
- 업무 API: FastAPI, SQLAlchemy, Alembic
- AI 서비스: FastAPI, 화자 비교·위변조·STT·NLP 어댑터
- 데이터베이스: PostgreSQL 운영 구성, SQLite 개발·테스트 호환
- 배포: Docker Compose, Nginx, 운영 점검 스크립트

## 2. 누적 개발 내용

### 인증 및 사용자

- 기존 이메일 중심 가입 화면을 휴대전화 OTP 기반 가입·로그인으로 전환했다.
- OTP 발송·확인, 가입용 검증 토큰, 액세스·회전형 새로고침 토큰을 구현했다.
- 신규 가입 시 본인 보호용 가족·보호 대상 문맥을 생성한다.
- 신규 계정 전환 시 이전 계정의 가족 상태와 늦게 도착한 세션 갱신 결과가 섞이지 않도록 정리했다.

### 가족 통화 보호

- 본인 보호와 부모님 등 가족 보호 흐름을 분리했다.
- 보호 대상과 보호 대상별 확인 가족을 등록·조회·삭제할 수 있다.
- 확인 가족은 등록 후 보호 대상 소유자의 승인, 재검증, 해지 상태를 갖는다.
- 가족 안전문구는 원문 대신 해시로 저장한다.

### 자료 확보 경로 확대

- SMS/링크 기반 등록 초대와 만료·재발송을 구현했다.
- 등록 기기 공개키와 서명을 이용한 단기 QR challenge를 구현했다.
- 직접 등록 시 liveness challenge를 적용했다.
- 외부 공유 파일은 MIME, 크기, 해시, 품질, 휴대전화 확인, 동의 단계를 거친다.
- 외부 자료는 검증 후에도 낮은 신뢰 등급에서 시작하고 만료 자료를 폐기할 수 있다.
- 음성·얼굴 자료의 길이·형식·크기·해시·동의 상태를 검증한다.

### 통화 분석 및 대응

- Android `CallScreeningService`가 수신 번호를 서버에 보내 통화 세션과 초기 판정을 생성한다.
- 서버 응답이 2.5초 안에 없으면 로컬 가족·위험번호 해시 캐시로 대체 판정한다.
- 번호 불일치, 화자 유사도, 합성음성 확률, 대화 위험 내용, 얼굴 점수, 가족 응답을 결합한다.
- 판정은 `ALLOW`, `VERIFY`, `RECALL`, `BLOCK`으로 구분한다.
- 동일 통화의 재판정은 순번이 증가하는 `RiskDecision`으로 보존한다.
- 확인 가족 푸시, 긴급 알림, 응답 반영, Android 조치 결과 보고를 구현했다.

### 배포 및 운영

- 개발·운영 Docker Compose, Nginx 설정, health/ready 점검과 운영 스크립트를 구성했다.
- Android debug/release 빌드와 운영 APK 배포 흐름을 구성했다.
- 운영 키스토어와 비밀번호는 Git 외부 보안 경로에서 관리한다.

## 5. 현재 구현 경계와 후속 과제

1. AI 화자 비교, 위변조 탐지, STT는 기본적으로 모의 어댑터를 사용하므로 상용 엔진 연결이 필요하다.
2. Web의 안전 통화·분석·차단·확인·기록 화면 일부는 시연 중심이며 Android 실통화 이벤트와의 실시간 동기화가 필요하다.
3. 개발 OTP와 개발용 초대 링크는 운영에서 SMS·딥링크 사업자 연계로 교체해야 한다.
4. 음성·얼굴 원본의 객체 저장소 암호화, 보존기간, 삭제 증적을 운영 인프라에 연결해야 한다.
5. 전화번호 원문, 통화 transcript, 푸시 토큰 등 민감정보의 암호화와 보존 정책을 강화해야 한다.
