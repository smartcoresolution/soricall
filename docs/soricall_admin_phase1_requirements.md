# SoriCall 관리자 페이지 1차 개발 기능 정의서

## 1. 문서 목적

본 문서는 SoriCall(안심소리 가족콜) 관리자 페이지에서 **우선 개발해야 할 기능만** 정의한다.

1차 개발의 목적은 다음과 같다.

- 어르신과 가족의 서비스 등록 상태를 관리한다.
- 가족 전화번호·음성·얼굴 등록 상태를 확인한다.
- 의심 통화의 분석 결과와 조치 이력을 추적한다.
- 가족 확인, 경고, 차단, 알림의 처리 결과를 관리한다.
- 개인정보 동의와 관리자 접근 이력을 안전하게 관리한다.

---

## 2. 1차 개발 범위 요약

| 우선순위 | 관리자 기능 | 핵심 목적 |
|---|---|---|
| P0 | 관리자 로그인 및 권한관리 | 허가된 관리자만 기능과 개인정보에 접근 |
| P0 | 운영 대시보드 | 사용자·가족 등록 및 위험 통화 현황 확인 |
| P0 | 사용자 관리 | 어르신·가족 계정과 연결 상태 관리 |
| P0 | 가족 인증정보 등록상태 관리 | 전화번호·음성·얼굴 등록 및 품질 상태 확인 |
| P0 | 통화 분석·위험 판정 이력 | 통화별 AI 분석 결과와 종합 위험도를 추적 |
| P0 | 경고·차단·알림 이력 | 위험 판정 이후 시스템 조치 결과 확인 |
| P0 | 가족 확인 응답 관리 | 실제 가족의 발신 여부 확인 결과 관리 |
| P1 | 신고·사건 관리 | 보이스피싱·오탐·미탐 신고 처리 |
| P0 | 개인정보 동의·파기 관리 | 동의 상태, 철회 및 데이터 파기 관리 |
| P1 | 기본 통계·보고서 | 운영 및 위험 통화 현황 집계·출력 |
| P0 | 관리자 작업·보안감사 로그 | 관리자 조회·변경·다운로드 행위 기록 |

`P0`는 서비스 개시 전에 반드시 구현하며, `P1`은 1차 개발 기간 안에 구현하되 P0 완료 후 개발한다.

```text
SoriCall 관리자
├─ 대시보드
├─ 사용자 관리
│  ├─ 어르신 사용자
│  └─ 가족 사용자
├─ 가족 인증정보 관리
├─ 통화 보안 관리
│  ├─ 통화 분석 이력
│  ├─ 경고·차단·알림 이력
│  └─ 가족 확인 응답
├─ 신고·사건 관리
├─ 개인정보 관리
│  ├─ 동의 현황
│  └─ 철회·파기 이력
├─ 통계·보고서
└─ 시스템 관리
   ├─ 관리자 계정·권한
   └─ 보안감사 로그
```

## 4. 기능별 상세 요구사항

### 4.1 관리자 로그인 및 권한관리 `[P0]`

#### 주요 기능

- 관리자 아이디·비밀번호 로그인
- 비밀번호 변경 및 초기화
- 로그인 실패 횟수 제한과 계정 잠금
- 일정 시간 미사용 시 자동 로그아웃
- 관리자 계정 생성·수정·사용중지
- 역할별 메뉴·조회·수정·다운로드 권한 설정
- 로그인 성공·실패 및 권한 변경 이력 기록

#### 1차 권한 구분

| 역할 | 주요 권한 |
|---|---|
| 최고관리자 | 전체 기능, 관리자 계정 및 권한 설정 |
| 운영관리자 | 사용자·통화·알림·신고 조회 및 운영 처리 |
| 고객지원 담당자 | 사용자 기본정보 조회, 문의·신고 처리 |
| 개인정보 담당자 | 동의·철회·파기 처리 및 접근 이력 점검 |
| 조회 전용 관리자 | 허용된 정보의 조회만 가능 |

#### 필수 보안 조건

- 비밀번호는 단방향 암호화하여 저장한다.
- 전화번호, 이름 등 개인정보는 목록 화면에서 마스킹한다.
- 음성·얼굴 원본은 기본적으로 재생·열람·다운로드할 수 없게 한다.
- 권한 없는 메뉴와 API 호출을 모두 차단한다.

### 4.2 운영 대시보드 `[P0]`

#### 핵심 현황 카드

- 전체 및 활성 어르신 수
- 등록 가족 수
- 가족 음성 등록 완료율
- 가족 얼굴 등록 완료율
- 오늘 분석 통화 수
- 정상·주의·위험·긴급 통화 수
- 자동 차단 통화 수
- 미응답 가족 확인 요청 수
- 미처리 신고 수

#### 최근 위험 통화 목록

- 통화 발생 시각
- 어르신 사용자
- 마스킹된 발신번호
- 추정 사칭 가족
- 종합 위험점수·위험등급
- 경고·차단 여부
- 가족 확인 결과
- 관리자 처리 상태

#### 조회 조건

- 오늘, 최근 7일, 최근 30일, 사용자 지정 기간
- 위험등급
- 처리 상태

---

### 4.3 사용자 관리 `[P0]`

#### 어르신 사용자 관리

- 사용자 번호, 성명, 휴대전화번호, 가입일
- 최근 접속일, 단말기, 운영체제, 앱 버전
- 보호 기능 활성 상태
- 등록 가족 수와 긴급 보호자
- 음성·얼굴 등록 완료 상태
- 개인정보 동의 상태
- 정상·정지·잠금·탈퇴 등 계정 상태

#### 가족 사용자 관리

- 가족 성명과 어르신과의 관계
- 휴대전화번호와 본인인증 상태
- 어르신 계정과의 연결 상태
- 음성·얼굴 등록 상태
- 가족 확인 요청 응답 여부와 최근 응답일
- 알림 수신 설정

#### 관리자 처리 기능

- 조건 검색 및 목록 조회
- 사용자 상세정보 조회
- 계정 잠금 해제 및 사용중지
- 보호 기능 활성·비활성 상태 확인
- 가족 연결 상태 확인 및 연결 해제 요청 처리
- 탈퇴·데이터 삭제 요청을 개인정보 관리로 이관

관리자가 사용자의 가족관계를 임의로 생성하거나 음성·얼굴 정보를 임의 변경하는 기능은 제공하지 않는다.

---

### 4.4 가족 인증정보 등록상태 관리 `[P0]`

#### 관리 항목

| 구분 | 주요 항목 |
|---|---|
| 기본정보 | 가족 식별번호, 성명, 관계, 전화번호, 본인인증, 어르신 연결 상태 |
| 음성정보 | 등록 여부, 샘플 수, 녹음 길이, 품질점수, 특징정보 생성 여부, 재등록 필요 여부 |
| 얼굴정보 | 등록 여부, 사진·영상 수, 품질점수, 얼굴 특징정보 생성 여부, 재등록 필요 여부 |
| 처리정보 | 등록일, 최근 수정일, 등록 요청자, 동의 상태 |

#### 관리자 처리 기능

- 전화번호·음성·얼굴의 등록 완료 여부 확인
- 품질 미달 및 처리 실패 사유 확인
- 사용자에게 재등록 요청 알림 발송
- 가족정보 삭제 요청을 개인정보 관리로 이관

음성·얼굴의 특징값과 품질 결과를 중심으로 관리하며 원본 열람은 최소화한다.

---

### 4.5 통화 분석 및 위험 판정 이력 `[P0]`

SoriCall 특허의 핵심 처리 과정을 통화별로 확인하는 기능이다.

#### 목록 항목

- 사건번호
- 통화 시작·종료 시각
- 통화를 받은 어르신
- 마스킹된 발신번호
- AI가 추정한 사칭 대상 가족
- 발신번호 일치 여부
- 가족 음성 유사도
- 합성·변조 음성 의심도
- 통화내용 위험도
- 얼굴 비교 결과(사용된 경우)
- 가족 확인 결과
- 종합 위험점수와 위험등급
- 최종 조치와 관리자 처리 상태

#### 상세 화면

- 분석 단계별 결과와 처리 시각
- 위험 판정 근거
- 경고·차단·가족 알림 이력
- 관련 가족 확인 응답
- 관련 신고·사건
- 관리자 처리 의견과 변경 이력

#### 검색 조건

- 기간, 사건번호, 사용자
- 위험등급: 정상·주의·위험·긴급
- 처리 결과: 연결·경고·차단·재통화 안내
- 가족 확인 결과
- 관리자 처리 상태

#### 관리자 조치

- 확인 전·확인 중·처리 완료 상태 변경
- 정상·오탐·실제 위험·미탐 분류
- 처리 의견 기록
- 신고·사건으로 전환

AI 판정값을 변경하는 경우 변경 전·후 값, 사유, 관리자, 변경 시각을 반드시 저장한다.

### 4.6 경고·차단·알림 이력 `[P0]`

#### 관리 항목

- 관련 사건번호
- 조치 유형: 화면 경고, 음성 경고, 통화 차단, 재통화 안내
- 알림 대상: 어르신, 가족, 긴급 보호자, 관리자
- 발송 채널: 앱 푸시, 문자 등
- 발송 내용과 발송 시각
- 성공·실패·수신·확인 여부
- 실패 사유와 재발송 결과

#### 관리자 처리 기능

- 조건별 이력 조회
- 발송 실패 건 재발송
- 관련 통화 분석 상세화면 이동

### 4.7 가족 확인 응답 관리 `[P0]`

#### 관리 항목

- 관련 사건번호와 확인 요청 시각
- 요청 대상 가족과 확인 방법
- 응답 시각
- 응답 결과
  - 제가 전화했습니다.
  - 제가 전화하지 않았습니다.
  - 잘 모르겠습니다.
  - 응답 없음
- 영상통화·얼굴 확인 결과(사용된 경우)
- 응답 이후 경고·차단·알림 조치

#### 관리자 처리 기능

- 미응답 요청 조회
- 응답 결과와 시스템 조치의 일치 여부 확인
- 비정상 처리 건을 신고·사건으로 전환

### 4.8 신고·사건 관리 `[P1]`

#### 신고 유형

- 실제 보이스피싱 또는 가족 사칭
- 합성·변조 음성 의심
- 정상 통화 오차단(오탐)
- 위험 통화 미탐
- 앱 오작동
- 개인정보 관련 신고

#### 관리 항목

- 신고번호, 신고자, 접수일시
- 관련 통화와 관련 사용자
- 신고 유형과 내용
- 피해 발생 여부와 피해금액
- 첨부파일
- 담당자, 처리 단계, 처리 의견
- 조치 결과와 종결일

#### 처리 단계

`접수 → 확인 중 → 가족 확인 → 조치 완료 → 종결`

### 4.9 개인정보 동의·철회·파기 관리 `[P0]`

#### 동의 항목

- 서비스 이용약관
- 개인정보 수집·이용
- 음성정보 수집·처리
- 얼굴정보 수집·처리
- 통화정보 및 통화내용 분석
- 가족정보 제공
- AI 학습 활용(선택)
- 제3자 제공(해당 시)

#### 관리 기능

- 사용자·가족별 동의 상태와 동의 버전 조회
- 동의일·철회일 및 동의 방법 조회
- 보유기간과 파기 예정일 관리
- 철회·탈퇴·삭제 요청 접수 및 처리
- 파기 대상, 처리자, 처리일, 처리 결과 기록
- 법적 보존 대상과 즉시 파기 대상을 구분

AI 학습 활용 동의는 선택 동의로 분리하며, 미동의 데이터를 학습데이터로 사용하지 않도록 통제한다.

- 개인정보는 권한에 따라 마스킹하거나 제외

### 4.11 관리자 작업 및 보안감사 로그 `[P0]`

#### 기록 대상

- 로그인 성공·실패와 로그아웃
- 사용자·가족정보 조회
- 음성·얼굴 관련 정보 접근
- 검색·상세조회·다운로드
- 계정 상태와 위험 판정 변경
- 알림 재발송
- 동의 철회·데이터 파기 처리
- 관리자 계정·권한 변경

#### 기록 항목

- 관리자 아이디와 역할
- 작업 일시
- 접속 IP와 기기·브라우저 정보
- 대상 사용자 또는 사건번호
- 작업 유형
- 변경 전·후 값
- 처리 사유와 성공·실패 결과

감사 로그는 일반 운영관리자가 수정하거나 삭제할 수 없도록 한다.

## 5. 공통 화면 요구사항

- 목록 화면은 검색, 필터, 정렬, 페이지 이동을 지원한다.
- 개인정보는 목록에서 마스킹하고 상세정보는 권한이 있는 관리자만 확인한다.
- 중요한 상태 변경에는 확인창과 변경 사유 입력을 적용한다.
- 모든 날짜·시간은 초 단위까지 저장하고 관리자 화면에는 한국 표준시로 표시한다.
- 오류 발생 시 사용자에게 원인과 재시도 방법을 안내하고 오류 로그를 남긴다.
- 대량 다운로드와 원본 데이터 다운로드는 기본적으로 제한한다.

## 7. 1차 개발 완료 기준

다음 조건을 충족하면 관리자 페이지의 1차 개발이 완료된 것으로 판단한다.

1. 관리자 역할별로 허용된 메뉴와 데이터만 접근할 수 있다.
2. 어르신·가족 계정과 가족 인증정보 등록 상태를 조회할 수 있다.
3. 통화별 발신번호, 음성, 합성 의심도, 통화내용, 얼굴, 가족 확인 결과와 종합 위험도를 추적할 수 있다.
4. 경고·차단·알림의 발송 및 처리 결과를 확인할 수 있다.
5. 오탐·미탐·실제 위험 건을 분류하고 처리 의견을 기록할 수 있다.
6. 개인정보 동의·철회·파기 상태를 조회하고 처리 이력을 남길 수 있다.
7. 주요 운영 통계를 기간별로 조회하고 파일로 출력할 수 있다.
8. 개인정보 조회와 주요 관리자 작업이 감사 로그에 누락 없이 기록된다.
9. 전화번호·이름 등 개인정보가 권한에 따라 마스킹된다.
10. 관리자 페이지의 주요 기능에 대한 기능시험과 권한시험을 통과한다.

## 8. 1차 개발 제외 항목

다음 기능은 1차 안정화 이후 2차 개발 대상으로 한다.

- 관리자 화면에서 AI 모델을 직접 배포하거나 이전 버전으로 복원하는 기능
- 위험점수 가중치와 기준값을 관리자가 직접 변경하는 기능
- 학습데이터 수집·라벨링·검수·학습 파이프라인 관리
- 실시간 서버·API·인프라 통합 관제
- 경찰 신고 시스템과의 자동 연계
- 지역·연령·관계별 고급 분석 및 BI 대시보드
- 공지사항, FAQ, 상담센터 등 전체 고객지원 CMS
- 통신사·외부기관용 별도 관리자 포털

## 9. 핵심 처리 흐름과 관리자 확인 지점

`수신 통화 발생 → 발신번호 확인 → 가족 음성 비교 → 합성·변조 의심 분석 → 통화내용 위험 분석 → 필요 시 얼굴·가족 확인 → 종합 위험도 산출 → 연결·경고·차단 → 가족 알림 → 관리자 이력 확인 및 사건 처리`

관리자 페이지는 위 과정의 각 분석 결과와 최종 조치가 하나의 사건번호로 연결되어 추적되도록 구현해야 한다.
