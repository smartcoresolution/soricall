# 안심소리 가족콜(SoriCall) 간략 UI 와이어프레임 설계서

작성일: 2026-07-06  
기준: 간략 서비스 순서도  
서비스 흐름: **가족 등록 → 전화 수신 → 가족 번호 확인 → 음성·얼굴·보호자 확인 → 차단 또는 안전 통화**

---

## 1. 설계 목적

안심소리 가족콜은 어르신에게 등록되지 않은 번호로 자녀·손자 등을 사칭하는 전화가 왔을 때, 이를 먼저 의심 전화로 안내하고 가족의 전화번호·음성·얼굴·보호자 확인을 통해 보이스피싱 피해를 줄이는 서비스이다.

본 문서는 간략 서비스 순서도에 맞춰 1차 MVP에서 바로 구현할 수 있는 **7개 핵심 스마트폰 화면**을 정의한다.

---

## 2. 전체 화면 구성

| 번호 | 화면명 | 주요 사용자 | 목적 |
|---:|---|---|---|
| 1 | 가족 등록 | 보호자/어르신 | 자녀·손자 전화번호, 얼굴, 음성 등록 |
| 2 | 어르신 홈 | 어르신 | 보호 상태 확인, 등록 가족에게 전화 |
| 3 | 전화 수신 | 어르신 | 상대방이 가족이라고 주장하는 전화 수신 |
| 4 | 의심 경고 | 어르신 | 등록되지 않은 번호일 때 즉시 경고 |
| 5 | 추가 확인 | 어르신/보호자 | 음성·얼굴·보호자 확인 |
| 6 | 결과 안내 | 어르신 | 안전 통화, 재확인, 차단 안내 |
| 7 | 보호자 알림 | 보호자 | 의심 전화 발생 시 보호자 응답 |

# 4. 화면별 상세 설계

## 4.1 화면 1: 가족 등록

### 목적

자녀·손자 등 실제 가족의 정보를 사전에 등록한다. 이 정보는 이후 의심 전화 판단의 기준 데이터가 된다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 가족 이름 | 예: 김민수 |
| 관계 | 아들, 딸, 손자, 손녀 |
| 전화번호 | 실제 가족 번호 |
| 얼굴 등록 | 사진 또는 카메라 촬영 |
| 음성 등록 | 짧은 문장 녹음 |
| 가족 추가하기 | 가족 등록 완료 또는 추가 등록 |

### 화면 문구

```text
가족 정보 사전 등록

자녀·손자 정보 등록
전화번호 등록
얼굴 등록
음성 등록

[가족 추가하기]
```

### 버튼 동작

| 버튼 | 동작 |
|---|---|
| 가족 추가하기 | 가족 정보 입력 화면 또는 등록 완료 처리 |
| 얼굴 등록 | 카메라/사진 등록 화면 이동 |
| 음성 등록 | 녹음 화면 이동 |

---

## 4.2 화면 2: 어르신 홈

### 목적

어르신이 현재 앱이 보호 중인지 확인하고, 등록된 가족에게 쉽게 전화할 수 있도록 한다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 보호 상태 문구 | “안심 보호 중입니다” |
| 가족 카드 | 가족 얼굴, 이름, 관계, 등록 상태 |
| 전화 버튼 | 등록 가족 번호로 전화 |
| 보호자 도움 요청 | 위험하거나 불안할 때 보호자에게 연결 |

### 화면 문구

```text
안심 보호 중입니다
등록된 가족 번호를 확인합니다

김민수 / 아들 / 등록 완료
[전화]

이지훈 / 손자 / 등록 완료
[전화]

[보호자에게 도움 요청]
```

### 설계 기준

- 가족 얼굴 또는 프로필 영역을 크게 표시한다.
- 전화 버튼은 한눈에 보이도록 초록색으로 표시한다.
- 어르신용 화면에서는 메뉴 수를 최소화한다.

---

## 4.3 화면 3: 전화 수신

### 목적

어르신에게 전화가 왔을 때, 상대방이 자녀·손자라고 주장하는 상황을 표시하고 등록된 가족 번호인지 확인한다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 전화 수신 아이콘 | 통화 상황 인지 |
| 상대방 주장 문구 | “상대방이 자녀·손자라고 주장” |
| 번호 확인 상태 | 등록된 가족 번호인지 확인 중 |
| 확인 결과 보기 | 확인 결과 화면으로 이동 |

### 화면 문구

```text
어르신에게 전화가 왔습니다

상대방이
자녀·손자라고 주장합니다

등록된 가족 번호인지 확인 중

[확인 결과 보기]
```

### 상태 분기

| 조건 | 다음 화면 |
|---|---|
| 등록된 가족 번호 | 결과 안내: 가족 확인됨 |
| 등록되지 않은 번호 | 의심 경고 화면 |
| 번호 확인 불가 | 추가 확인 화면 |

---

## 4.4 화면 4: 의심 경고

### 목적

등록된 가족 번호가 아닌 번호로 가족을 사칭하는 경우, 어르신에게 즉시 위험 행동을 막도록 안내한다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 경고 아이콘 | 큰 주황/빨강 경고 표시 |
| 핵심 문구 | 등록된 가족 번호가 아님 |
| 금지 행동 안내 | 돈, 계좌, 비밀번호, 앱 설치 금지 |
| 전화 끊기 버튼 | 가장 중요한 행동 버튼 |

### 화면 문구

```text
등록된 가족 번호가 아닙니다
가족 사칭 전화일 수 있습니다

돈을 보내지 마세요
계좌번호를 말하지 마세요
앱 설치를 하지 마세요

[전화 끊기]
```

### 설계 기준

- 위험 상황에서는 버튼을 1~2개로 제한한다.
- “위험 점수”보다 “지금 무엇을 해야 하는가”를 먼저 보여준다.
- 확정 표현을 피하고 “의심”, “가능성” 중심으로 표현한다.

---

## 4.5 화면 5: 추가 확인

### 목적

의심 전화에 대해 가족 여부를 한 번 더 확인한다.

### 확인 방식

| 확인 방식 | 설명 |
|---|---|
| 음성 확인 | 등록된 가족 목소리와 비교 |
| 얼굴 확인 | 화상 또는 얼굴 이미지와 비교 |
| 보호자 확인 | 실제 가족에게 전화 여부 확인 요청 |

### 화면 문구

```text
가족인지 추가 확인합니다

음성 확인
등록된 목소리와 비교

얼굴 확인
화상으로 얼굴 확인

보호자 확인
가족에게 확인 요청

[보호자에게 확인 요청]
```

### 상태 문구

| 상태 | 문구 |
|---|---|
| 확인 중 | 가족인지 확인하고 있습니다 |
| 음성 일치 | 등록된 가족 목소리와 비슷합니다 |
| 음성 불일치 | 등록된 목소리와 다를 수 있습니다 |
| 얼굴 일치 | 등록된 얼굴과 비슷합니다 |
| 얼굴 불일치 | 얼굴 확인이 필요합니다 |
| 보호자 미응답 | 저장된 가족 번호로 다시 전화하세요 |

---

## 4.6 화면 6: 결과 안내

### 목적

확인 결과에 따라 어르신에게 최종 행동을 안내한다.

### 결과 유형

| 결과 | 색상 | 문구 | 행동 |
|---|---|---|---|
| 가족 확인됨 | 초록 | 가족 확인됨 | 안전 통화 |
| 확인 어려움 | 노랑 | 확인 어려움 | 가족 번호로 다시 전화 |
| 가족 아님 의심 | 빨강 | 가족 아님 의심 | 통화 차단 |
| 보호자 알림 | 주황/빨강 | 보호자 알림 | 보호자 응답 대기 |

### 화면 문구

```text
가족 확인됨 → 안전 통화
확인 어려움 → 가족 번호로 다시 전화
가족 아님 의심 → 통화 차단

[보호자 알림]
```

### 설계 기준

- 최종 결과는 색상과 문구를 함께 사용한다.
- 어르신에게는 숫자 점수보다 행동 문구를 우선한다.
- “확인 어려움”인 경우에는 저장된 가족 번호로 재통화하도록 안내한다.

---

## 4.7 화면 7: 보호자 알림

### 목적

의심 전화 발생 시 보호자에게 즉시 알림을 보내고, 보호자가 사실 여부를 응답할 수 있도록 한다.

### 주요 UI 요소

| 요소 | 설명 |
|---|---|
| 의심 전화 알림 | 부모님에게 의심 전화 발생 |
| 위험 사유 | 미등록 번호, 가족 사칭 주장 |
| 전화번호 | 마스킹 표시 |
| 보호자 응답 | 맞음 / 아님 / 모르겠음 |
| 부모님께 전화하기 | 저장된 번호로 재연결 |

### 화면 문구

```text
부모님에게 의심 전화가 왔습니다

미등록 번호
자녀라고 주장한 전화
010-****-7788

[내가 전화한 것 맞음]
[내가 전화한 것 아님]
[모르겠음]
[부모님께 전화하기]
```

### 보호자 응답 결과

| 보호자 응답 | 어르신 화면 안내 |
|---|---|
| 내가 전화한 것 맞음 | 가족으로 확인되었습니다 |
| 내가 전화한 것 아님 | 가족 사칭이 의심됩니다. 전화를 끊으세요 |
| 모르겠음 | 저장된 가족 번호로 다시 전화하세요 |
| 부모님께 전화하기 | 보호자가 직접 어르신에게 전화 |

# 5. 공통 디자인 기준

## 5.1 색상

| 용도 | 색상 | HEX |
|---|---|---|
| 기본 배경 | 아이보리 | `#FFF8EC` |
| 메인 컬러 | 청록 | `#25A99A` |
| 정상 | 초록 | `#2EAD6B` |
| 전화/정보 | 파랑 | `#2D7CC1` |
| 확인/판단 | 보라 | `#7B61A8` |
| 주의 | 노랑 | `#F5C542` |
| 의심 | 주황 | `#F2994A` |
| 차단 | 빨강 | `#E84B3C` |

## 5.2 글자 크기

| 요소 | 어르신 화면 권장 크기 | 보호자 화면 권장 크기 |
|---|---:|---:|
| 화면 제목 | 28~34px | 24~28px |
| 핵심 안내 | 26~32px | 20~24px |
| 본문 | 22~26px | 16~18px |
| 버튼 | 24~28px | 16~18px |
| 보조 문구 | 18~22px | 14~16px |

## 5.3 버튼 원칙

| 상황 | 버튼 개수 | 예시 |
|---|---:|---|
| 정상 통화 | 1~2개 | 전화 받기, 추가 확인 |
| 의심 전화 | 1~2개 | 전화 끊기, 보호자 알림 |
| 결과 안내 | 1~2개 | 다시 전화, 보호자 알림 |
| 보호자 화면 | 3~4개 가능 | 맞음, 아님, 모르겠음, 전화하기 |

# 6. 간략 서비스 흐름

```text
가족 정보 사전 등록
        ↓
어르신에게 전화 수신
        ↓
등록된 가족 번호인가?
   ┌────┴────┐
   │         │
  예        아니오
   │         │
정상 통화   의심 전화 표시
             ↓
        추가 확인
   음성 확인 / 얼굴 확인 / 보호자 확인
             ↓
        결과 안내
안전 통화 / 재확인 / 통화 차단 / 보호자 알림
```

# 9. 결론

안심소리 가족콜의 1차 UI는 복잡한 인증 절차보다 **등록된 가족 번호가 아닌 번호로 가족을 사칭할 때 즉시 의심하고, 어르신이 저장된 가족 번호로 다시 확인하도록 안내하는 것**에 집중해야 한다.

가장 중요한 화면 문구는 다음과 같다.

```text
지금 전화를 끊고
저장된 가족 번호로 다시 전화하세요.
```
